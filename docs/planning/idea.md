# Talby Workforce Raw Idea

I want to create a system to work as a team of agents to solve problems.

I want the system to be composed of the following components:

- **Task Management**: An integrated task management system to interact with external sources to read and track projects and task statuses. This will serve as a queuing system for the agents to pick up tasks and work on them. I want to make integrations with popular task management tools like Trello, Linear, OpenProject, Github Issues, Worklenz, etc. So we need an abstraction layer to interact with these tools in a unified way. The system will not have a task management implementation of its own, at least not in the beginning, but will rely on these external tools to manage tasks and projects.
- **Memory System**: A memory system to store information about the agents, their interactions, and the tasks they are working on. This will allow the agents to learn from their experiences and improve their performance over time. A candidate to implement this system is Honcho. The memory system abstraction must allow to ingest conversations and sessions between multiple agents and humans, and must be able to answer natural language questions about the information stored in the memory.
- **Execution System**: A system to start workflows and agents to execute tasks. This system will be responsible for orchestrating the agents and ensuring that they are working on the right tasks at the right time. It will also be responsible for handling any errors or issues that may arise during the execution of tasks. A candidate to implement this system is Kestra.
- **Git Integration**: A git integration to allow the agents to interact with code repositories. This will allow the agents to read and write code, as well as track changes and collaborate with other agents and humans. Some repositories are about specifications, others are about code, and others are about End to End tests. The system must be able to interact with all of them in a unified way, and must be able to understand the context of the repositories and the tasks at hand.
- **Agent System**: A system to create and manage agents, skills, prompts, evaluations, and other components that will allow the agents to perform their tasks effectively. This system will be responsible for creating and managing the agents, as well as providing them with the necessary tools and resources to perform their tasks.
- **Other Integrations**: There must be other types of integrations, exposed to agents through MCPs or other means, to allow the agents to interact with other tools and services that may be relevant to their tasks.
- **User Interface**: A user interface to allow humans to interact with the system, monitor the progress of tasks, and provide feedback to the agents. This will be an important component to ensure that the system is user-friendly and accessible to non-technical users.
- **API**: An API to allow external systems to interact with the system and access its functionalities. This will allow other systems to integrate with the system and leverage its capabilities to solve problems and perform tasks.
- **CLI**: A command-line interface to allow users to interact with the system through terminal commands. This will provide an alternative way to access the system's functionalities, especially for advanced users and automation scripts.
